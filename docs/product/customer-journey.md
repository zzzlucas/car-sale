# 客户主链路

## 目标用户

目标用户是需要办理车辆报废回收的个人客户或车辆所有人，他们最关心的是三件事：

- 平台是否正规、可信
- 流程是否省心、步骤是否清楚
- 提交之后能不能持续看到进度

## `v1` 客户路径

### 1. 首页建立信任
- 展示“正规、合法、省心”的核心价值
- 强调认证资质、流程透明、进度可查
- 引导用户进入“立即估价 / 预约回收”

### 2. 提交车辆信息
- 用户填写车辆基础信息
- 用户补充必要资料和照片
- 系统生成初步估价和预约申请

### 3. 等待平台联系
- 用户提交预约后，进入待联系或待确认阶段
- 平台通过电话或后台流程继续确认细节

### 4. 查看进度
- 用户能持续看到订单当前阶段
- 时间线展示要和后台真实状态一致
- 关键节点需要尽量解释清楚，例如是否已安排拖车、是否在办理注销

### 5. 获得帮助
- 用户可以查看流程说明和常见问题
- 用户可进入客服入口咨询当前订单或流程问题

## `v1` 客户端主要页面

- 首页
- 车辆估价评估
- 订单进度追踪
- 报废流程指南
- 我的
- 在线客服

## 客户侧体验原则

- 术语尽量简单，不让客户理解后台行业黑话
- 关键动作前后都给到明确反馈
- 进度状态和客服入口要容易找到
- 资质与合规信息应持续强化信任感

## 当前不纳入客户端 `v1` 的能力

- 在线支付
- 复杂营销活动
- 面向多企业的 marketplace
- 车管所结果的自动同步展示
