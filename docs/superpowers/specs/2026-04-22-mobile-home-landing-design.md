# 移动端首页落地页化设计

## 背景

当前客户首页同时承担了品牌展示、估价入口和半表单采集三种角色，导致两个明显问题：

- 顶部固定 `bar` 与首屏标题重复表达品牌，首屏显得拥挤。
- “输入车牌号”输入框没有承接到实际估价流程，用户输入没有真实业务收益。

在当前项目阶段，首页更适合回到单一职责：建立信任并把用户送进估价页。

## 目标

- 将客户首页收敛为纯转化落地页。
- 删除无实际承接的车牌输入框。
- 去掉首页固定顶栏，让首屏本身承担品牌表达。
- 保留现有底部导航，不改动整体路由结构。
- 继续沿用当前主题色和 Tailwind 变量，不引入新的重型设计体系。

## 设计结论

### 1. 首页角色

首页只承担“获客转化”职责，不承担信息采集职责。

信息采集统一放在估价页完成，首页只展示：

- 价值主张
- 信任补充
- 主 CTA
- 核心优势
- 服务流程
- 常见问题 / 联系客服

### 2. 首屏结构

首屏改为无顶栏海报式结构，包含：

- 小号品牌标识或业务标签
- 主标题：突出“专业、合法、省心”
- 补充文案：强调正规资质、注销代办、上门拖车
- 主按钮：`立即估价 / 预约回收`
- 3 个信任徽章：合法合规、免费拖车、进度可查

不再出现任何输入控件。

### 3. 内容层级

首页按以下顺序组织：

1. Hero：品牌承诺 + 主 CTA
2. 核心优势：四项能力说明
3. 服务流程：四步说明
4. 常见问题：三条高频问题
5. 联系支持：联系客服入口

### 4. 视觉方向

- 视觉 thesis：浅底、深青主色、柔和高光，像“正规服务落地页”而不是后台表单。
- 内容 plan：Hero -> 优势 -> 流程 -> FAQ -> CTA。
- interaction thesis：
  - 首屏主卡片保留轻微景深和柔光背景
  - CTA 保持明确对比，不叠加次级干扰动作
  - 列表区块用轻边框和留白建立秩序，不堆卡片噪音

## 范围说明

本轮只调整客户首页展示结构，不变更：

- 估价页字段结构
- 路由入口
- 底部导航信息架构
- 后端接口

## 风险

- 首页去表单后，估价页入口的重要性更高，CTA 文案必须足够明确。
- 若后续要做“快捷估价”能力，应直接设计成可承接的轻流程，而不是重新塞回首页伪输入框。

## 验证标准

- 首页源码中不再出现固定顶栏和车牌输入文案。
- 首页仍保留到 `/customer/valuation` 的主入口。
- 首页出现 FAQ 与联系客服入口。
- 移动端页面可以正常通过类型检查、测试和构建。
